# Copilot 使用说明

## 使用目标
通过 GitHub Copilot 加速样板代码编写、辅助生成 AI 策略框架、提升结构一致性与命名规范。

## 触发高质量建议的策略
1. 预先写出函数签名与 JSDoc 注释（描述参数与返回值）。
2. 在文件顶部用注释说明模块职责与设计思想。
3. 先列出伪代码或关键步骤，让 Copilot 自动补全细节。
4. 生成后人工审查：语义准确性、边界条件覆盖、可读性。必要时进行重构。
5. 提交信息中标注受 Copilot 影响的片段，例如：`feat(ai): adapt copilot heuristic suggestion`。

## 示例注释驱动
```js
// 目标：实现中级 AI 策略：赢棋 > 阻挡 > 中心 > 角 > 边
// 输入：board(Array[9]), aiPlayer('X'|'O')
// 输出：index(0-8)
```
让 Copilot 在此基础上补足条件与返回语句。

## 建议采纳标准
- 保留清晰逻辑的自动补全；删除不必要嵌套与重复判断。
- 对含有 "魔法数字" 的建议添加常量或注释。
- 所有非纯逻辑（如 DOM 操作）确保不与状态管理耦合。

## 风险与规避
| 风险 | 描述 | 规避 |
|------|------|------|
| 过度依赖 | 直接接受复杂函数 | 拆分函数 + 逐段验证 |
| 隐藏 Bug | 忽略边界条件 | 添加空数组、终局状态测试用例规划 |
| 风格不统一 | 不同片段命名风格差异 | 建立统一命名规范与模块职责说明 |

## 后续改进
- 结合测试框架让 Copilot 生成单元测试草稿。
- 使用问题导向提示：例如“生成更易读的 Minimax 实现并加剪枝”。
